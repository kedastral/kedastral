# Kedastral Operator Demo (kind)

A one-command, end-to-end demo of Kedastral's **operator** running predictive
autoscaling on a local [kind](https://kind.sigs.k8s.io/) cluster. Unlike the
[`simple-app`](../simple-app/) demo (minikube + flag-based manifests), this one drives
everything through the operator: you declare a `DataSource` and `ForecastPolicy`, and
Kedastral runs the forecast loop and generates the KEDA `ScaledObject` for you.

## What it brings up

- **kind** cluster (`kedastral-demo`)
- **KEDA** (event-driven autoscaling)
- **Prometheus** (minimal, static scrape config)
- **Sample app** + **load generator** (reused from `examples/simple-app`, run without
  the optional PostgreSQL backend)
- **Kedastral** installed via Helm in **operator mode**
- A **DataSource** + **ForecastPolicy** that forecasts the app's request rate

## Prerequisites

- `docker`, `kind`, `kubectl`, `helm`

## Run

```bash
cd examples/kind-operator
./setup.sh
```

The script builds the Kedastral, app, and load-generator images, loads them into kind,
installs everything, and applies the demo `DataSource`/`ForecastPolicy`. It takes a few
minutes on first run (image builds + KEDA install).

## Watch it work

```bash
# Policy status, the generated ScaledObject, the HPA KEDA creates, and the pods
kubectl get forecastpolicy,scaledobject,hpa,pods -n kedastral-demo

# Policy detail: current/desired replicas, last forecast time, Ready condition
kubectl describe forecastpolicy simple-app -n kedastral-demo

# Forecaster logs
kubectl logs -f -n kedastral-demo -l app.kubernetes.io/component=forecaster
```

The `ScaledObject` named `simple-app` is **owned by** the `ForecastPolicy` — deleting
the policy stops its forecast loop and garbage-collects the ScaledObject:

```bash
kubectl delete forecastpolicy simple-app -n kedastral-demo
kubectl get scaledobject -n kedastral-demo   # gone
```

## Try different load patterns

```bash
kubectl set env deploy/load-generator -n kedastral-demo PATTERN=double-peak
# patterns: constant, hourly-spike, business-hours, sine-wave, double-peak
```

## Dashboards

Prometheus scrapes the app and both Kedastral components, so you can import
[`deploy/grafana/kedastral-dashboard.json`](../../deploy/grafana/) into any Grafana
pointed at this Prometheus (`kubectl port-forward -n kedastral-demo svc/prometheus 9090:9090`).

## Tear down

```bash
./cleanup.sh    # deletes the kind cluster (prompts for confirmation)
```

## How it differs from `examples/simple-app`

| | `kind-operator` (this) | `simple-app` |
|---|---|---|
| Cluster | kind | minikube |
| Kedastral config | ForecastPolicy/DataSource CRDs (operator) | forecaster/scaler flags |
| ScaledObject | generated by the operator | hand-written manifest |
| Database | none | PostgreSQL |
